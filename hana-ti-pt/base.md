## 5. 3-Level 시스템 (핵심 업적)

### Level 1 — ML 팩터 모델 → 사내 첫 AI 공모펀드 런칭
- XGBoost + 재무 데이터 기반 S&P500 Growth 공모펀드 (설정액 34억원)
- 시그널 생성 → 종목 선정 → 포트폴리오 구축 → 실행, 엔드투엔드 파이프라인
- 16개월 실전 운용 경험

### Level 2 — LLM 기반 멀티스타일 포트폴리오 파이프라인 + 기관 검증
- Level 1 기저 ML 모델 유지 + LLM으로 다양한 투자 목적함수(라벨) 자동 생성
- 동일 피처 공간에서 상이한 리스크-수익 목표의 복수 전략 변형 체계적 탐색
- 리스크-리턴 평면 매핑 → 효율적 프론티어 시각화
- 고객의 리스크 선호에 따른 인터랙티브 포트폴리오 선택 체계
- **다올자산운용 전략 공급 MOU 체결** — 외부 기관 검증

### Level 3 — Agentic Alpha Factory (팀 리더)
- 멀티에이전트 시스템(Claude Agent SDK) 기반 자동화 리서치 엔진
- 전략 가설 생성 → 코드 구현 → 백테스트 → 평가 → 자가 진화
- 알파 합성 전략 및 팩터 중립화 모델 개발
- **일 500+ 전략 아이디어 자동 생성·검증**
- 팀 리더(Team Lead)로서 설계·총괄

## 8. AI-native 리서치 역량

### 핵심 차별점
- AI를 "보조 도구"로 끼워넣는 것 ≠ AI를 "연구의 중심 아키텍처"로 설계하는 것
- 전자: 기존 워크플로우에 AI 추가
- 후자: AI 특성에 맞춰 워크플로우 자체를 재설계

### 할루시네이션 제어 시스템
- 테스트 기반 개발 (TDD)
- 멀티에이전트 병렬 교차 리뷰
- 사람이 효율적으로 검증할 수 있는 보고서 형태 출력 설계
- 아키텍처 수준의 방어 (프롬프트가 아님)

### 생산성
- LLM 코딩으로 이동 중/수면 시간 활용 → 하루 90%+ 코드 개발 투입 가능
- 단, 부작용(할루시네이션)을 이해하고 시스템으로 방어

## 10. 하나금융TI에서의 비전

### 왜 하나금융TI인가
1. **연구 깊이 확장**: 금융 도메인 특화 연구 인력이 집중된 조직, 동료 리서처와 심도 있는 탐구
2. **그룹 데이터 인프라**: 은행·증권·보험 멀티도메인 데이터 → Alpha Factory의 입력 다양성 ↑ → 인간이 예상 못 한 팩터 조합 발견 확률 ↑

### 입사 후 목표
- **단기 (6개월)**: 기존 리서치 프레임워크 학습 + Agentic 파이프라인 PoC
- **중기 (1~2년)**: Alpha Factory를 그룹 인프라에 안착, 리서치 생산성 10x
- **장기**: 하나금융그룹 = 한국 금융권 AI-native 리서치 표준

### 킬러 한 줄
"개별 운용사에서는 접근 불가한 그룹 데이터에 Agentic Factory를 올리면, 인간이 예상 못 한 팩터 조합이 발견될 확률이 근본적으로 올라갑니다."
